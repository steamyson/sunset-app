<task type="auto">
  <name>Task 1: app.json — register expo-image-picker plugin and declare export compliance</name>
  <files>app.json</files>
  <action>
Two changes to app.json:

1. Add expo-image-picker to the `plugins` array (after the existing "expo-web-browser" entry):
```json
[
  "expo-image-picker",
  {
    "photosPermission": "Dusk needs photo library access to set your profile photo."
  }
]
```
This registers NSPhotoLibraryUsageDescription on iOS. Without it, accessing the photo library via expo-image-picker crashes on iOS because the native permission string is missing from Info.plist.

2. Add `infoPlist` to the existing `ios` object:
```json
"ios": {
  "supportsTablet": true,
  "bundleIdentifier": "com.akivagroener.dusk",
  "infoPlist": {
    "ITSAppUsesNonExemptEncryption": false
  }
}
```
Apple requires this declaration. The app only uses standard HTTPS and iOS Keychain (both exempt categories), so the value is `false`.
  </action>
  <verify>
    <automated>node -e "const c=require('./app.json'); const p=c.expo.plugins; const hasIP=p.some(x=>Array.isArray(x)?x[0]==='expo-image-picker':x==='expo-image-picker'); const hasEnc=c.expo.ios.infoPlist?.ITSAppUsesNonExemptEncryption===false; console.log('image-picker plugin:', hasIP, '| encryption decl:', hasEnc); if(!hasIP||!hasEnc) process.exit(1);"</automated>
  </verify>
  <done>app.json has expo-image-picker in plugins with photosPermission, and ios.infoPlist.ITSAppUsesNonExemptEncryption is false.</done>
</task>
