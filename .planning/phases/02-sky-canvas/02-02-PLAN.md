**5. Add a useEffect to run the overlay expansion animation when overlayTarget is set:**

```typescript
useEffect(() => {
  if (!overlayTarget) return;
  const { room } = overlayTarget;
  const screenW = W;
  const screenH = H;

  Animated.parallel([
    Animated.spring(overlayLeft, { toValue: 0, tension: 200, friction: 14, useNativeDriver: false }),
    Animated.spring(overlayTop, { toValue: 0, tension: 200, friction: 14, useNativeDriver: false }),
    Animated.spring(overlayWidth, { toValue: screenW, tension: 200, friction: 14, useNativeDriver: false }),
    Animated.spring(overlayHeight, { toValue: screenH, tension: 200, friction: 14, useNativeDriver: false }),
    Animated.spring(overlayRadius, { toValue: 16, tension: 200, friction: 14, useNativeDriver: false }),
  ]).start(({ finished }) => {
    if (finished) {
      const unread = unreadRooms.has(room.code);
      router.push(`/room/${room.code}${unread ? "?unread=true" : ""}`);
      // Dismiss overlay after a short delay to let navigation take over
      setTimeout(() => {
        overlayOpacity.setValue(0);
        setOverlayTarget(null);
      }, 300);
    }
  });
}, [overlayTarget]);
```

Spring config: `tension: 200, friction: 14` (faster, more zoom-like than the standard 120/8 -- Claude's discretion per CONTEXT.md). `useNativeDriver: false` is mandatory -- animating layout properties.
